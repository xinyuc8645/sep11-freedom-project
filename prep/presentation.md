# Presentation Plan

## Hook
* Maybe we can start off with explaining the theme of our game and the concept behind it. We can put gifs/titles of giving hints of what our game is going to be about and the theme revolving around it. Additionally we can maybe start off with this, "have you ever been so bored playing an easy game that you are looking for an hardcore game to play and cure your boredom? Well this is the exact game for you then" Something like that

## Product
* Do an quick casual demo 1-2 minutes
* Explain the concepts like not too complicated; for example explain the coins system, explain the key movements, death system, brickwall and others

## Process
* Explain an certain part of the code within our game
* For example: for me I'll explain wanting to make the sprite turn the other way and challenges I've come across while in the process
* Like creating two sprite to mimic the direction
* Then using phaser documentation to find solutions
* Then explain setFlipx; how it contributes to our sprite in turning the other way
* Explain the true or false statement within
* Then my partner will explain his process and challenges for his code of the part of the game

## Conclusion
* Takeaways; There should be 3 main takeaways
* Takeaway one --> only use AI for assistance not for getting the answer directly
* Takeaway two --> Learn something frist from experience, then do the process. So that way you won't go into something completely blind
* Takeaway three --> communication is key; always communicate with your teammate and keep in touch with them
